1. Overview

This landing page is a modern, fully responsive AI-powered learning platform template. It includes multiple sections, smooth animations, a demo request dialog, and comprehensive styling.

Key Features:

  • Fully responsive design (mobile, tablet, desktop)
  • Smooth scroll animations using Intersection Observer API
  • Interactive demo request modal/dialog
  • Modern gradient design with turquoise brand color (#40E0D0)
  • Pricing tables with multiple plans
  • Testimonials section with customer reviews
  • Mobile-friendly navigation with hamburger menu
  • Form validation and submission handling

3. Getting Started

Quick Start (No Server Required)

  1. Open index.html directly in your web browser
  2. Make edits to HTML, CSS, or JavaScript files
  3. Refresh the browser to see changes

Using a Local Server (Recommended)

For better development experience, use a local server:

Option 1: VS Code Live Server

  1. Install "Live Server" extension in VS Code
  2. Right-click index.html
  3. Select "Open with Live Server"

Option 2: Python Server

# Navigate to project folder
cd innovate-tech-landing

# Python 3
python -m http.server 8000

# Open browser to http://localhost:8000

Option 3: Node.js Server

# Install http-server globally
npm install -g http-server

# Run server
http-server -p 8000

8.3 Integrating with Backend/API

Example: Send form data to your server: 

demoForm.addEventListener("submit", async (e) => {
    e.preventDefault();
    
    const formData = new FormData(demoForm);
    const data = Object.fromEntries(formData);
    
    try {
        const response = await fetch('https://your-api.com/submit', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
            },
            body: JSON.stringify(data)
        });
        
        if (response.ok) {
            alert('Thank you! We will contact you soon.');
            closeModalFunc();
            demoForm.reset();
        }
    } catch (error) {
        alert('Error submitting form. Please try again.');
    }
});

9. Animations & Effects

9.1 Scroll Animations

Animations are triggered using Intersection Observer API in script.js (~lines 857-916): 

function initScrollAnimations() {
    const sections = document.querySelectorAll('section');
    
    // Animation patterns for each section
    const animationPatterns = [
        null,           // Hero - no animation
        'fade-scale',   // How It Works
        'slide-left',   // Benefits
        'slide-right',  // Demo
        'fade-up',      // Testimonials
        'fade-scale',   // For Schools
        'fade-up',      // CTA
        'slide-left'    // Pricing
    ];
}

9.2 Available Animation Types

Animation Class Effect Usage
fade-up Fades in while moving up General sections
fade-scale Fades in while scaling up Feature highlights
slide-left Slides in from the left Alternating sections
slide-right Slides in from the right Alternating sections

12. Troubleshooting

12.1 Modal/Dialog Not Opening

Issue: Clicking CTA buttons doesn't open the modal

Solution:

  1. Check browser console for JavaScript errors (F12)
  2. Verify onclick="openModal()" is on the button
  3. Ensure script.js is loaded
  4. Check that dialog element has id="demo-modal"

12.2 Animations Not Working

Issue: Scroll animations don't trigger

Solution:

  1. Verify browser supports Intersection Observer (IE11 doesn't)
  2. Check console for errors
  3. Ensure initScrollAnimations() is being called
  4. Verify animation classes are in styles.css

12.3 Mobile Menu Not Working

Issue: Hamburger menu doesn't open/close

Solution:

  1. Check that hamburger button has correct event listeners
  2. Verify .nav-menu.active CSS is present
  3. Clear browser cache

12.4 Styling Issues

Issue: CSS not applying correctly

Solution:

  1. Hard refresh browser (Ctrl+Shift+R or Cmd+Shift+R)
  2. Check styles.css is linked in <head>
  3. Verify no CSS syntax errors
  4. Check browser developer tools for overridden styles
  5. Ensure correct CSS specificity

12.5 Form Submission Issues

Issue: Form doesn't submit or gives errors

Solution:

  1. Check preventDefault() is called in form handler
  2. Verify all required fields have required attribute
  3. Check network tab in DevTools for API errors
  4. Ensure CORS is configured if submitting to external API
